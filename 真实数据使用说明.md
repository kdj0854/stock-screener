## 股票筛选与策略回溯工具 - 使用说明

## 数据架构

本项目使用两套数据库并行运行：

| 数据库 | 类型 | 内容 |
|--------|------|------|
| `market_data.db` | SQLite | 股票基本信息、PE/PB/市值等估值数据 |
| `stock_data.db`  | SQLite | 回测交易记录、RL训练记录 |
| 价格数据库        | MySQL  | A 股历史日线价格 |

---

## 环境要求

- Python 3.8+
- Node.js 18+
- MySQL（含 `dbbardata` 历史行情表）

---

## 安装步骤

### 1. 安装 Python 依赖

```bash
cd backend
pip install -r requirements.txt
```

主要依赖：
- `baostock` — A 股免费数据源（行情/估值/财务）
- `fastapi` / `uvicorn` — 后端 API 框架
- `sqlalchemy` — ORM 操作 SQLite
- `pymysql` — 连接 MySQL 价格数据库
- `pandas` / `numpy` — 数据处理与夏普率计算
- `optuna` — 贝叶斯参数优化（RL 框架）

### 2. 配置 MySQL 连接

通过环境变量配置数据库连接：

```bash
set MYSQL_HOST=127.0.0.1
set MYSQL_PORT=3306
set MYSQL_USER=root
set MYSQL_PASSWORD=你的密码
set MYSQL_DATABASE=stock_price
```

也可在 `backend/config.py` 中直接修改默认值。

### 3. 安装前端依赖

```bash
cd frontend
npm install
```

## 数据同步流程

首次使用或需要更新数据时，按以下顺序操作：

### 第一步：数据刷新（同步价格）

在「股票筛选」界面点击 **「数据刷新」** 按钮。

- 从 baostock 拉取 A 股历史日线数据（后复权），写入 MySQL 价格数据库
- 自动增量更新：仅补充上次同步后的新数据，已有数据不重复写入
- 数据范围：从各股最新入库日期到当日，新股从 2015-01-01 开始
- 进度条实时显示当前处理股票和累计新增条数
- 节假日/周末无交易数据属正常现象

### 第二步：同步基本面

在「股票筛选」界面点击 **「同步基本面」** 按钮。

- 从 baostock 获取股票名称、行业分类
- 获取真实 PE(TTM)、PB(MRQ)、PS(TTM)
- 市值计算：`收盘价 × 总股本（万股，来自 query_profit_data）/ 10000`，单位亿元
- 计算近 1 年年化夏普率
- 全量同步约 5000 只股票，预计耗时 30 分钟以上，建议后台运行
- 支持增量更新，重复点击不会重复创建记录

> ⚠️ **注意**：市值筛选功能依赖「同步基本面」的数据，请先完成同步再使用筛选。

## 功能说明

### 股票筛选

筛选条件（均可自定义）：

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 最小市盈率 | PE 下限，过滤已亏损标的 | 0 |
| 最大市盈率 | PE 上限，寻找低估标的 | 30 |
| 最大市值 | 单位亿元，聚焦中小盘 | 60 |
| 连续年数 | PE 需连续满足条件的年数（当年+历年） | 3 |

筛选完成后，符合条件的股票自动勾选到左侧列表，供回测使用。

### 策略回测

基于「跌买止盈」策略回测历史表现：

| 参数 | 说明 |
|------|------|
| 跌买阈值 | 股价从过去 N 天最高点下跌超过该比例时买入 |
| 盈利目标 | 买入后浮盈达到该比例时卖出 |
| 回溯周期 | 计算历史最高价的时间窗口（天） |

回测结果包含：胜率、平均收益率、累计收益率曲线、买卖点分析率、交易明细。

### 参数优化训练

使用 **Optuna TPE 贝叶斯优化**自动搜索最优参数组合：

- 探索范围：跌买阈值 5%~40%，盈利目标 1%~20%
- 筛选条件：胜率 ≥ 85% 才视为有效轮次
- 奖励函数：Σ（平仓收益率 × 10 万元）
- 可视化：奖励曲线 + 参数探索散点图（金色三角标注最优轮次）

## 常见问题

### Q：数据刷新显示 0 条新增，正常吗？

A：正常。若当前数据库已是最新（如今日为周末/节假日），则没有新数据可补充，显示 0 条属预期行为。

### Q：同步基本面时间太长怎么办？

A：可以先点击筛选，已有数据的股票可正常筛选。同步基本面可以分批进行，每次点击都是增量更新，不会重复处理已同步的记录。

### Q：市值筛选结果不准确？

A：市值基于 baostock `query_profit_data` 接口的真实总股本计算，必须完成「同步基本面」后才准确。若显示 `-` 或数值异常，说明该股票尚未同步，请先执行同步。

### Q：MySQL 连接失败？

A：检查以下几点：
1. MySQL 服务是否已启动
2. 环境变量中的密码是否正确
3. 数据库名与 `config.py` 中配置是否一致
4. 查看后端启动时的错误日志

### Q：baostock 登录失败？

A：检查网络连接，baostock 需要访问外网（`baostock.com:9999`）。若频繁失败，可能是服务器维护，稍后重试。
